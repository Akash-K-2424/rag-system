================================================================================
                        ARCHITECTURE OVERVIEW
================================================================================

RAG PIPELINE:
  User Query
      ↓
  [Query Understanding]
      ↓
  [Embedding Generation] → SentenceTransformers
      ↓
  [Vector Database Search] → ChromaDB (Top-5)
      ↓
  [Context Re-ranking] → MMR Algorithm
      ↓
  [Answer Generation] → Gemini API
      ↓
  [Citations] → Document + Page Number
      ↓
  Answer + Citations + Confidence

AGENT ORCHESTRATION (5 Tools):
  1. tool_ingest_documents() - PDF parsing & chunking
  2. tool_embed_chunks() - Embedding generation
  3. tool_vector_search() - Vector similarity search
  4. tool_rerank_context() - MMR re-ranking
  5. tool_generate_answer_with_citations() - Answer generation

HALLUCINATION CONTROL (3 Mechanisms):
  1. Context-only prompting
  2. Confidence threshold filtering
  3. Citation extraction & verification

                        API ENDPOINTS
================================================================================

GET /health
  Response: {status, vector_db_ready, embedding_model_ready}

POST /upload
  Input: PDF file
  Response: {filename, chunks_created, total_tokens, status}

POST /chat
  Input: {query: string}
  Response: {answer, citations, confidence, retrieved_chunks}

================================================================================
                        CONFIGURATION
================================================================================

Environment Variables:
  • GEMINI_API_KEY (required)
  • BACKEND_HOST (default: 0.0.0.0)
  • BACKEND_PORT (default: 8000)
  • CHROMA_DB_PATH (default: ./chroma_db)
  • EMBEDDING_MODEL (default: sentence-transformers/all-MiniLM-L6-v2)
  • CHUNK_SIZE (default: 400)
  • CHUNK_OVERLAP (default: 80)
  • TOP_K_RETRIEVAL (default: 5)
  • CONFIDENCE_THRESHOLD (default: 0.5)

================================================================================
                        PERFORMANCE
================================================================================

Latency:
  • Document ingestion: 2-5 seconds per page
  • Query processing: 3-8 seconds
    - Embedding: ~0.5s
    - Retrieval: ~0.5s
    - Re-ranking: ~1s
    - Generation: ~2-5s

Storage:
  • Per chunk: ~1.5KB
  • 1000 chunks: ~1.5MB

Scalability:
  • ChromaDB: 100K+ chunks
  • SentenceTransformers: Batch processing
  • Gemini API: Rate limits apply

================================================================================
                        DEPLOYMENT
================================================================================

LOCAL DEVELOPMENT:
  Backend:  cd backend && python main.py
  Frontend: cd frontend && npm run dev

PRODUCTION (Render + Vercel):
  Backend:  Push to GitHub → Create Render Web Service
  Frontend: Push to GitHub → Import in Vercel

DOCKER:
  Build:  docker build -t rag-system-backend backend/
  Run:    docker run -p 8000:8000 -e GEMINI_API_KEY=... rag-system-backend
